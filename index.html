Frequently Asked Questions

What databases does MAIGI support?

+
MAIGI is PostgreSQL-native. We support PostgreSQL 13+. MariaDB and other databases are in Phase 2 roadmap. PostgreSQL's native MVCC and JSON support make it ideal for intelligent context deduplication.

How much can we actually save on token costs?

+
Average savings is 90% through context deduplication. For enterprise sales teams running 10+ agents on the same customer data: $12K/month → $1.2K/month. For support teams: $1.20/ticket → $0.15/ticket. Real savings depend on your agent concurrency and context reuse patterns.

How does MAIGI handle multi-agent consistency?

+
MAIGI uses ACID-guaranteed transactions via PostgreSQL with saga orchestration for distributed operations. If Agent A modifies context, Agent B sees the consistent state immediately. No race conditions. Full audit trail of all changes.

Can we self-host MAIGI or is it SaaS only?

+
Both! MAIGI is available as a hosted SaaS option OR self-hosted via Docker. Deploy in your own VPC in minutes. No vendor lock-in. You own your data and infrastructure.

What's the setup time and learning curve?

+
MAIGI is designed for rapid deployment. Local setup with docker-compose takes ~5 minutes. If you know PostgreSQL and Python/Go, integration with your agents is straightforward. Most teams go from zero to production in 2-3 days.

Is MAIGI production-ready for enterprise use?

+
MAIGI (launching Q2 2026) is production-ready for 10-100 concurrent agents. Enterprise features (monitoring, scaling, SLA support) ship in Phase 2. Built on PostgreSQL—the same database powering Fortune 500 companies.
